# <a name="events-generated-by-the-odbc-driver-manager"></a>ODBC 드라이버 관리자에 의해 생성 된 이벤트
> [!IMPORTANT]  
>  Visual Studio Analyzer에 대 한 지원 (Visual Studio Analyzer 이전 버전의 Visual Studio에만 포함 되었습니다.) Windows 8 부터는 제거 되었습니다. 다른 방법은 문제 해결 메커니즘 BID 추적을 사용 합니다.  
  
 Visual Studio Analyzer 시작 단추를 클릭할 때 ODBC 드라이버 관리자에 의해 생성 된 이벤트 등록 됩니다. 도구 자체는 시스템에서 정의한 이벤트와 사용자 지정 이벤트를 만들 수 있는 기능을 제공 합니다. 이벤트에 대 한 자세한 내용은 참조는 *Visual Studio 분석기 참조 가이드* 설명서의 Visual Studio 도구 모음 내에서.  
  
|Visual Studio Analyzer 이벤트|Description|  
|----------------------------------|-----------------|  
|**호출**|모든 ODBC API 항목에 생성 됩니다.|  
|**ReturnException**|반환 코드는 SQL_ERROR 경우 모든 ODBC API 반환 시 생성.|  
|**ReturnNormal**|반환 코드 SQL_ERROR 되지 않으면 모든 ODBC API 반환 시 생성 합니다.|  
|**연결 시작**|연결을 시작 합니다. ODBC 드라이버 관리자 드라이버의 연결 Api를 호출 하는 경우 발생 합니다.|  
|**연결 완료**|연결이 완료 되었음을; 나타냅니다. 드라이버의 연결 Api ODBC 드라이버 관리자를 반환 하는 경우 발생 합니다.|  
|**연결 끊기 시작**|ODBC 드라이버 관리자는 드라이버를 호출할 때 생성 **SQLDisconnect** 함수입니다.|  
|**연결 끊기 완료**|경우에 생성 되는 드라이버의 **SQLDisconnect** 함수는 ODBC 드라이버 관리자를 반환 합니다.|  
|**QuerySend**|ODBC 드라이버 관리자는 드라이버를 호출할 때 생성 **SQLPrepare**, **SQLExecute**, **SQLExecDirect** 같은카탈로그함수뿐만아니라함수,**SQLTables** 및 **SQLColumns**합니다.|  
|**QueryResult**|드라이버는 결과 집합 쿼리를 포함 하는 함수에 대 한 ODBC 드라이버 관리자를 반환 하는 경우 발생 합니다.|  
|**TransactionStart**|응용 프로그램이 sql_autocommit_off를 SQL_ATTR_AUTOCOMMIT의 값을 설정 또는 응용 프로그램을 성공적으로 호출한 후 생성 **SQLEndTran**합니다.|  
|**TransactionCommit**|응용 프로그램이 호출 하는 경우에 생성 **SQLEndTran** 된 로컬 트랜잭션을 커밋합니다.|  
|**TransactionRollback**|응용 프로그램이 호출 하는 경우에 생성 **SQLEndTran** 로컬 트랜잭션을 롤백할 수 있습니다.|  
|**JoinDTC**|응용 프로그램 Distributed Transaction Coordinator (DTC)를 조인 하는 경우 발생 합니다.|  
|**LeaveDTC**|응용 프로그램 Distributed Transaction Coordinator (DTC)를 벗어날 때 발생 합니다.|
