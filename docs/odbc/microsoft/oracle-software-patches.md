# <a name="oracle-software-patches"></a>Oracle 소프트웨어 패치
> [!IMPORTANT]  
>  이 기능은 나중 버전의 Windows에서 제거 됩니다. 새 개발 작업에서는 이 기능을 사용하지 않도록 하고, 현재 이 기능을 사용하는 응용 프로그램은 수정하세요. Oracle에서 제공 하는 ODBC 드라이버를 사용 하십시오.  
  
 여러 Microsoft 제품 및 기술에 Microsoft ODBC Driver for Oracle Microsoft OLE DB Provider for Oracle에서는 인터넷 정보를 포함 하 여 적절 하 게 작동에 필요한 Oracle 서버 제품 및 해당 클라이언트 구성 요소에 대 한 패치를 서비스 (IIS) 구성 요소 서비스 (또는 Microsoft Transaction Server, Windows NT를 사용 하는 경우) 등입니다.  
  
> [!NOTE]  
>  다음 지침에서는 Oracle FTP 사이트는 변경 될 수 있으므로 완전히 정확 하 게 아닐 수 있습니다.  
  
### <a name="to-download-the-oracle-software-patches"></a>Oracle 소프트웨어 패치를 다운로드 하려면  
  
1.  Oracle ftp.oracle.com에서 공용 FTP 사이트에 연결 합니다. 사용자 ID가 "익명"이 고 암호는 전자 메일 주소입니다.  
  
2.  다음 디렉터리로 이동: /server/wgt_tech/server/windowsNT 합니다.  
  
3.  Windows 95, Windows 98 및 Windows NT/Windows 2000에 대 한 가장 관련성이 높은 패치를 다운로드 하려면 Oracle 버전에 대 한 하위 디렉터리로 이동-7.3 또는 8.0 합니다. 두 개의 하위 디렉터리는 /73patchsets 및 /80patchsets입니다.  
  
4.  두 SQL, Oracle의 네트워크 기술에 대 한 패치를 다운로드 하려면 * Net 또는 다음 디렉터리로 이동 Net8: / 네트워크입니다.  
  
 웹 브라우저에서이 FTP 사이트에 액세스 하는 작동 하지 않을 수 있습니다. 문제가 발생 하면 "전통적인" FTP 클라이언트를 사용 하거나 DOS 명령 프롬프트를 사용 하십시오.  
  
> [!NOTE]  
>  Oracle 현재 버전에서 버그를 수정 하 고 다음에 소프트웨어 패치를 사용 하 여 이전 버전으로 retrofits, 때문에 사용할 수 있는 최신 패치를 다운로드 하는 것이 좋습니다. Oracle Server 클라이언트 구성 요소에 대 한 경우 특히 유용합니다. 이 패치를 설치 하는 방법에 대 한 질문이 있으면 Oracle 지원에 문의 합니다.
