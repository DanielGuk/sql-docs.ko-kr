# <a name="what39s-new-in-ssma--for-oracle-oracletosql"></a>기능 &#39;의 새로운 Oracle (OracleToSQL) 용 SSMA
이 항목에서는 각 릴리스의 Oracle 변경에 대 한 SSMA를 나열 합니다.  

## <a name="may-2016"></a>2016 년 5 월  
Oracle 용 SSMA의 2016 년 5 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  

1.  SQL Server 2016에 대 한 지원 추가
2.  SQL Server 임시 테이블에 추가 된 Oracle 플래시 백 보관 테이블 변환
3.  SQL Server 정책 개체가 (Oracle에 대 한 행 수준 보안)를 변환 하는 Oracle VPD 정책으로 추가 된 변환
4.  Oracle에 대 한 초기 로드의 줄어든된 시간
5.  향상 된 파서 및 해결 프로그램
6.  .NET 2.0에 대 한 설치 관리자 검사를 제거합니다.
7.  .NET 3.5에서에서.Net 4.0 업데이트 된 확장 팩 종속성
8.  프로젝트"저장" 고정 및 SSMA 콘솔에 대 한 프로젝트 열기 명령
9.  SSMA 콘솔에 대 한 고정된 "securepassword" 명령
10. 초기 로드에 대 한 개체의 계산 고정
11. Oracle에 대 한 문자 데이터 형식의 변환 고정
12. 전역 설정에서 수정 된 버그
  
## <a name="march-2016"></a>2016 년 3 월  
Oracle 용 SSMA의 2016 년 3 월 미리 보기 릴리스는 다음과 같은 변경을 포함 되어 있습니다.  
  
1.  SQL Server 2016으로 마이그레이션 지원  
  
2.  마이그레이션 Oracle 행 수준 보안 (을 몇 가지 제한 사항이)에 대 한 지원  
  
3.  마이그레이션에 대 한 지원 메모리에는 Oracle SQL Server 열 저장소 테이블  
  
## <a name="january-2016"></a>2016 년 1 월  
2014 년 1 월 Oracle 용 SSMA의 유지 관리 릴리스에 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  클러스터형 인덱스에 대 한 지원 추가  
  
2.  고정된 느린 Oracle 스키마 쿼리 (RFC 5076207)  
  
3.  고정 콘솔에서 Azure에 연결  
  
4.  SSMA (RFC 5706203)에 추가 된 보기 로그 메뉴 항목  
  
5.  추가 된 원격 분석  
  
## <a name="july-2014"></a>2014 년 7 월  
Oracle 용 SSMA의 2014 년 7 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  Azure SQL DB에 대 한 지원  
  
2.  Azure SQL DB를 지원 하기 위해 스키마로 이동 하는 확장 팩 기능  
  
3.  Oracle 구체화 된 뷰에 대 한 지원  
  
4.  SQL Server 2014 메모리에 대 한 지원을 최적화 된 테이블  
  
5.  성능 향상 개체 10k 사용 하 여 데이터베이스에 대 한 테스트  
  
6.  향상 된 다 수의 개체를 처리 하기 위한 UI  
  
7.  "잘 알려진" LOB 스키마의 강조 표시  
  
8.  변환 속도 향상  
  
9. UI에서 개체 수를 표시 합니다.  
  
10. 25%를 초과 하 여 보고서 크기 감소  
  
11. 구문 분석 되지 않은 구문에 대 한 향상 된 오류 메시지입니다.  
  
## <a name="april-2014"></a>2014 년 4 월  
Oracle 용 SSMA의 2014 년 4 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  MS SQL Server 2014의 지원이 추가 되었습니다.  
  
2.  Oracle 12 및 쿼리 최적화의 지원이 추가 되었습니다.  
  
3.  Azure에 변환에 대 한 수정 된 버그입니다.  
  
4.  IE 10에서 보이지 않는 보고서 페이지에 대 한 수정 된 버그입니다.  
  
## <a name="january-2012"></a>2012 년 1 월  
Oracle 용 SSMA의 2012 년 1 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   지원 RowType 및 RecordType 입력된 매개 변수를 NULL 기본값입니다.  
  
## <a name="july-2011"></a>2011 년 7 월  
Oracle 용 SSMA의 2011 년 7 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   에 대 한 Oracle 시퀀스는 변환에 대 한 지원 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] "Denali" 시퀀스 생성기입니다.  
  
-   향상 된 오류 데이터 마이그레이션 중에 보고 합니다.  
  
-   예약 된 단어를 사용 하 여 문의 향상 된 변환 합니다.  
  
-   향상 된 함수에서 날짜 값의 암시적 변환입니다.  
  
## <a name="april-2011"></a>2011 년 4 월  
Oracle 용 SSMA의 2011 년 4 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   지 원하는 통합된 "SSMA에 대 한 Oracle" 제품 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2005 년 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 2008 및 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] "Denali"입니다.  
  
-   연결 하 고로 마이그레이션에 대 한 지원 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] "Denali"입니다.  
  
-   클라이언트 쪽 데이터 마이그레이션 엔진을, 데이터의 병렬 마이그레이션을 지 원하는 향상 되었습니다.  
  
-   향상 된 데이터 마이그레이션 성능 단순 및 대량 로그 복구 모델.  
  
-   SSMA (v4.0 및 v4.2)의 이전 버전에서 만든 프로젝트의 이전 버전과 호환성.  
  
-   V5.0 제품 수 있습니다. Oracle 용 SSMA 이전 버전의 SSMA (v4.0 및 v4.2)와 나란히 (SxS)를 설치 합니다.  
  
-   사용자 정의 형식 (하위 유형, VARRAY, 중첩 테이블, 개체 테이블 및 개체 보기 포함) 및 특수 한 오류 메시지와 함께 PL/SQL 블록에서 사용법을 보고에 대 한 지원.  
  
## <a name="july-2010"></a>2010 년 7 월  
Oracle 용 SSMA의 2010 년 7 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   SQL Server 2008 r 2로 마이그레이션에 대 한 지원  
  
-   명령줄 실행에 대 한 새 SSMA 콘솔 응용 프로그램  
  
-   서버 쪽 및 클라이언트 쪽 데이터 마이그레이션 엔진 둘 다 사용 하 여 데이터 마이그레이션에 대 한 지원  
  
-   데이터 마이그레이션에 대 한 문 "사용자 지정 선택"에 대 한 지원  
  
-   Oracle 11g R2 마이그레이션에 대 한 지원  
  
## <a name="june-2008"></a>2008 년 6 월  
Oracle 용 SSMA의 2008 년 6 월 릴리스는 다음과 같은 변경을 포함 되어 있습니다.  
  
-   평가 보고서의 향상 된 기능으로 완료 되었습니다. 동의어에 대 한 추가 정보, 원시 소스를 구문 분석할 수 개체, 패널 및 SQL Server 로고 제거 및 레이아웃 지 속성을 포함합니다.  
  
-   개체 변환의 향상 된 기능:  
  
    -   패키지 DBMS_LOB, DBMS_SQL 변환을 추가 합니다.  
  
    -   조인 수정 변환 합니다.  
  
    -   컬렉션 및 레코드 변환, 대부분의 경우 각 필드에 대 한 별도 변수를 통해 릴리스 레코드의 변환 지금 수정 합니다.  
  
    -   레코드 및 컬렉션 구현의 개선 되었습니다.  
  
    -   기간 이동 집계 함수를 추가 합니다.  
  
    -   롤업/큐브 절을 추가 합니다.  
  
    -   NEXTVAL/CURVAL 성능이 향상 됩니다.  
  
    -   SET 절에서 그룹화 열 그룹화 집합 및 ID 그룹화가 추가 되었습니다.  
  
    -   MERGE 문 추가 합니다.  
  
    -   새 날짜/시간 형식 및 추가 된 CLR 데이터 형식으로 변환 레코드와 컬렉션을 지원 합니다.  
  
-   테스터의 새로운 기능이 추가 되었습니다. 테스터를 사용 하 여 개체로 테스트할 수 이제 테이블, 테스트 사례에 몇 가지 테스트 가능한 개체의 호출 순서를 변경할 수 있습니다, 그리고 사용자 절차와 레코드 및 컬렉션 매개 변수로 사용 하 여 함수를 테스트 하 고 반환 값 및 분석기 확인에 추가 된 종속성 테이블에만 사용 합니다.  
  
## <a name="august-2007"></a>2007 년 8 월  
Oracle 용 SSMA의 2007 년 8 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   새 테스터 구성 요소를 사용 하 여 생성, 관리 및 변환 된 SQL 코드를 확인 하는 테스트 사례를 실행할 수 있습니다.  
  
-   Oracle 하위 유형, 컬렉션 및 로컬 모듈의 변환 SQL 변환기에 추가 되었습니다.  
  
-   새 동기화 기능에는 특정 개체는 SQL Server 데이터베이스와 동기화 할 수 있습니다.  
  
-   새 변환 옵션을 추가 합니다.
