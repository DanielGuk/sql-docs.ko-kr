# <a name="what39s-new-in-ssma-for-accessaccesstosql"></a>기능 &#39;의 새로운 Access(AccessToSQL) 용 SSMA
이 항목에서는 각 릴리스에 대 한 액세스 변경에 대 한 SSMA를 나열 합니다.  
  
## <a name="may-2016"></a>2016 년 5 월  
Access 용 SSMA의 2016 년 5 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  SQL Server 2016에 대 한 공식 지원
2.  .NET 2.0에 대 한 설치 관리자 검사를 제거합니다.
3.  프로젝트"저장" 고정 및 SSMA 콘솔에 대 한 프로젝트 열기 명령
4.  SSMA 콘솔에 대 한 고정된 "securepassword" 명령
5.  초기 로드에 대 한 개체의 계산 고정
6.  고정된 테이블 데이터 액세스를 위해 UI 탭에 대 한 로드
7.  전역 설정에서 수정 된 버그 
   
## <a name="march-2016"></a>2016 년 3 월  
Access 용 SSMA의 2016 년 3 월 미리 보기 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  SQL Server 2016으로 마이그레이션 지원  
   
## <a name="january-2016"></a>2016 년 1 월  
Access 용 SSMA의 2016 년 1 월 유지 보수 릴리스는 다음과 같은 변경을 포함 되어 있습니다.  
  
1.  기본값은 GUID 필드 (RFC 3894811)에 대 한 고정된 잘못 된 함수  
  
2.  SQL 데이터베이스 (Azure) (RFC 4919573)에 레코드 가져오기에 대 한 고정된 중지  
  
3.  SSMA (RFC 5706203)에 추가 된 보기 로그 메뉴 항목  
  
4.  추가 된 원격 분석  
  
## <a name="july-2014"></a>2014 년 7 월  
Access 용 SSMA의 2014 년 7 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  향상 된 Azure SQL DB 코드 변환  
  
2.  Azure SQL DB를 지원 하기 위해 스키마로 이동 하는 확장 팩 기능  
  
3.  성능 향상 개체 10k 사용 하 여 데이터베이스에 대 한 테스트  
  
4.  향상 된 다 수의 개체를 처리 하기 위한 UI  
  
5.  (되므로 변환 과정에서 무시) "잘 알려진" LOB 스키마의 강조 표시  
  
6.  변환 속도 향상  
  
7.  UI에서 개체 수를 표시 합니다.  
  
8.  25%를 초과 하 여 보고서 크기 감소  
  
9. 구문 분석 되지 않은 구문에 대 한 향상 된 오류 메시지입니다.  
  
## <a name="april-2014"></a>2014 년 4 월  
Access 용 SSMA의 2014 년 4 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
1.  MS SQL Server 2014의 지원이 추가 되었습니다.  
  
2.  Azure에 변환에 대 한 수정 된 버그입니다.  
  
3.  IE 10에서 보이지 않는 보고서 페이지에 대 한 수정 된 버그입니다.

## <a name="may-2007"></a>2007 년 5 월  
Access 용 SSMA의 2007 년 5 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   작업 그룹 정책을 사용 하는 Access 데이터베이스에 대 한 지원입니다.

-   SQL Server 형식이 SQL 모드를 지정 하는 사용자가 입력 한 주석에 대 한 지원.  
  
-   개체 변환에서 개선 되었습니다.  
  
## <a name="november-2006"></a>2006 년 11 월  
Access 용 SSMA의 2006 년 11 월 릴리스는 다음과 같은 변경 내용이 포함 되어 있습니다.  
  
-   에 대 한 액세스에서 단일 데이터베이스의 마이그레이션을 통해 안내해 주는 새 데이터베이스 마이그레이션 마법사 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)]합니다.  
  
-   Access 데이터베이스를 변환 하는 새 변환, 로드 및 마이그레이션 명령으로 변환된 된 개체를 로드 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)], 고 데이터를 마이그레이션하 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] 한꺼번에 합니다.  
  
-   향상 된 쿼리 마이그레이션입니다. 변환에는 더 쿼리 뷰를 선택 하는 이제 마이그레이션을 쿼리 합니다. 자세한 내용은 참조 [Access 데이터베이스 개체가 변환](http://msdn.microsoft.com/en-us/e0ef67bf-80a6-4e6c-a82d-5d46e0623c6c)합니다.  
  
-   이제 테이블 및 인덱스 속성을 편집할 수 있습니다는 [!INCLUDE[ssNoVersion](../../includes/ssnoversion_md.md)] **테이블** 탭 합니다.  
  
-   새 전역 설정:  
  
    -   편집기 창에서 줄 번호를 표시 하도록 선택할 수 있습니다.  
  
    -   SSMA 중복 개체를 바꿀 것인지 묻는 메시지를 구성 하거나 항상 또는 never 스키마 변환 하는 동안 중복 된 개체를 바꿀 수 있습니다.  
  
-   새 변환 옵션을 사용 하면 SSMA 복잡 한 쿼리를 와일드 카드를 포함 하는 경우 경고를 표시 하는 경우 지정할 수 있습니다.
